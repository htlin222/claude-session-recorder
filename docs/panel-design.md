# explainshell 側欄設計（決策與演化）

右側那塊 [explainshell](https://github.com/idank/explainshell) 風格的命令拆解側欄，
是經過幾輪 brainstorm + smoke test 才定案的。記錄選了什麼、為什麼。

## 為什麼要側欄
原本畫面右半邊一直是空的（終端輸出靠左、行短）。拿來放同步的「重點提示」
增加學習理解。**關鍵約束**：底部已有字幕（旁白逐字），側欄必須顯示
**字幕沒有的、互補的**東西，否則就是同一資訊塞兩遍。
→ 三層各司其職：**語音=為什麼／情境**、**字幕=同步文字**、**側欄=命令逐字語法**。

## 內容 = explainshell 拆解 + 一句話重點
不真的跑 explainshell 那個工具（要 MongoDB + 解析 man page，殺雞用牛刀）；
**複刻它的視覺**（命令逐段拆解 + 註解），註解內容手寫（更準更精煉）。
每段再配一句話重點橫幅。

## 動態 = 隨打字浮現（appear-on-type）
選項比較：
- 隨旁白逐項高亮（教學最強，但需逐詞時間戳、約束腳本）
- **隨打字浮現**（跟「打字」節拍天然同步，不需逐詞時間戳）← 選這個
- 整塊靜態出現（最簡單，但沒有「跟著走」的引導感）

## 版面 = 回顯命令 + 短樁，靠顏色＋順序連結
最初學 explainshell 從命令裡的 token 拉**長垂直連線**到下方註解。
**踩坑**：命令是橫向展開、註解是縱向堆疊，靠右的 token 連線會**穿過**
前面註解的文字 → 線跟字交錯，比終端還亂（窄欄尤其）。

**修法**（smoke test 驗證過）：
- **砍掉長連線**，token 下只留 16px **短樁**（不可能交錯）。
- 靠**顏色 + 順序**連結：token 在命令裡上色高亮，註解行用**同色左條 + 同色標籤**，
  命令左→右 = 註解上→下，一一對應。
- 顏色語義：`ord` 綠（一般 flag）、`star` 桃（該段關鍵 flag）、`path` 紫（路徑）。

對應的 variant 在 smoke test 出了 5 版（語義+短樁 / 無樁 / 素色 / 危險紅 / 寬鬆大字），
最後選 **A：語義三色 + 短樁**。

## 長命令 = 換行，不縮字
`--exclude` 那段命令很長，在 720px 側欄回顯：
- 早期用「自動縮小字號」塞進一行 → 字太小。
- **定案：固定字號，超長就換到第 2 行**（`overlay.py` 的 `wrap_cmd`）。
  token 的高亮 / 短樁要**按所在行**定位（`char_xy` 把字元索引映射到 (line, col)）。
- 註解只標**重點 token**：重複的 `--exclude` 合併說明、`A/ C/` 路徑簡述，
  不是每個 token 都拉一條，免得擠。

## 畫布 = 1920×1080，左右合成
終端渲染成左側 1200px（VHS `Set Width 1200`），ffmpeg `pad` 到 1920×1080，
右側 720px 疊面板。好處：終端文字物理上進不了面板區，零重疊；
中間一條細分隔線。（終端 1200px + 字號 26 也讓最長輸出行不再換行。）

## 字幕：外掛，不燒錄
最後一版要求字幕**不要燒進畫面** → `overlay.py` 移除字幕燒錄，
只輸出外掛 `rsync-demo-final.srt`（播放器自動載入同名字幕）。
側欄是圖形疊加（保留），字幕是文字軌（外掛）。

## 想改側欄
- 文案／配色：改 `build.py` 的 token 註解或 `overlay.py` 的 `ROLE` 顏色，
  **不必重渲染 VHS**，跑 `overlay.py` ~10s 出片。
- 版面（字號 / 行距 / 換行寬度）：`overlay.py` 的 `CMD_SIZE` / `ROW_H` / `CPL`。
